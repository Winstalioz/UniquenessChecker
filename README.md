# ИНСТРУКЦИЯ ПО РАЗВЕРТЫВАНИЮ И ЭКСПЛУАТАЦИИ ПРОГРАММНОГО КОМПЛЕКСА «УНИКДОС»

## 1 Структура программного пакета

Исходный код и сопроводительные материалы системы проверки уникальности текстовых работ «УникДос» организованы в виде следующей структуры каталогов:

```text
UniquenessChecker/
├── plantuml/                       — Исходные файлы UML-диаграмм (PlantUML)
│   ├── 01_usecase.puml             — Диаграмма вариантов использования (Раздел 1.2)
│   ├── 02_classes.puml             — Диаграмма классов (Раздел 1.2)
│   ├── 03_sequence.puml            — Диаграмма последовательности (Раздел 1.2)
│   └── 04_deployment.puml          — Диаграмма развёртывания (Раздел 1.2)
│
├── sql/
│   └── create_database.sql         — Полный DDL/DML-скрипт развертывания СУБД
│
└── UniquenessChecker/              — Проектная папка среды Visual Studio (C# WPF)
    ├── UniquenessChecker.sln       — Файл решения программного комплекса
    └── UniquenessChecker/
        ├── App.config              — Конфигурационный файл (строка подключения)
        ├── Models/Models.cs        — Описание доменных моделей данных
        ├── Data/DatabaseRepository.cs — Репозиторий доступа к данным (ADO.NET)
        └── Views/                  — Графические интерфейсы пользователя
            ├── LoginWindow.xaml    — Интерфейс авторизации
            ├── AdminWindow.xaml    — Интерфейс автоматизированного рабочего места администратора
            ├── ClientWindow.xaml   — Интерфейс преподавателя и студента
            └── ReportWindow.xaml   — Интерфейс аналитической отчетности

```

## 2 Развертывание реляционной базы данных

### 2.1 Системные требования

Для развертывания серверной части системы требуется установленная СУБД Microsoft SQL Server (версии Express, Developer или Standard) и графическая среда администрирования SQL Server Management Studio (SSMS).

### 2.2 Порядок выполнения скрипта развертывания

1. Запустите среду **SQL Server Management Studio** и выполните подключение к целевому экземпляру сервера.
2. В главном меню выберите пункт **File → Open → File...** и укажите путь к файлу `sql/create_database.sql`.
3. Нажмите клавишу **F5** (или кнопку «Execute» на панели инструментов) для инициализации транзакции.
4. При успешном завершении операции в панели «Messages» отобразится следующее системное уведомление:
```text
>>> База данных uniqueness_db создана.
>>> ВСЁ ГОТОВО! Таблиц: 7 | Индексов: 5 | Представлений: 4 | Процедур: 6

```



```

## 4 Конфигурация строки подключения к СУБД

Для установления соединения между клиентским приложением и сервером баз данных необходимо отредактировать конфигурационный файл **`UniquenessChecker/App.config`**, изменив параметры в секции `<connectionStrings>`:

```xml
<add name="UniquenessDb"
     connectionString="Data Source=.\SQLEXPRESS;Initial Catalog=uniqueness_db;Integrated Security=True;" />

```

### Модификации строки подключения в зависимости от топологии сети

* **Конфигурация 1: Локальный экземпляр SQL Server Express (стандартная установка)**
```text

```



Data Source=.\SQLEXPRESS;Initial Catalog=uniqueness_db;Integrated Security=True;

```
* **Конфигурация 2: Стандартный именованный экземпляр сервера**
  ```text
  Data Source=.\MSSQLSERVER;Initial Catalog=uniqueness_db;Integrated Security=True;

```

* **Конфигурация 3: Использование SQL-аутентификации (при отключенной авторизации Windows)**
```text
Data Source=.\SQLEXPRESS;Initial Catalog=uniqueness_db;User Id=sa;Password=ВашТекущийПароль;

```



```
* **Конфигурация 4: Удаленный сервер в локальной сети организации**
  ```text
  Data Source=192.168.1.100,1433;Initial Catalog=uniqueness_db;Integrated Security=True;

```

## 5 Установка внешних зависимостей (NuGet)

Управление внешними библиотеками осуществляется через менеджер пакетов NuGet. Восстановление зависимостей происходит автоматически при первой сборке проекта. В случае возникновения ошибок компиляции, установку криптографического пакета BCrypt необходимо произвести вручную через консоль:

1. Перейдите в меню: **Tools → NuGet Package Manager → Package Manager Console**.
2. Выполните команду:
```text

```



Install-Package BCrypt.Net-Next

```

## 7 Методика визуализации UML-диаграмм (PlantUML)

Архитектурные модели системы описаны на декларативном языке PlantUML. Для их конвертации в графический формат PNG предусмотрено несколько вариантов.

### Вариант 1: Использование веб-сервиса (без установки ПО)
1. Скопируйте текстовое содержимое любого файла с расширением `.puml` из каталога `plantuml/`.
2. Перейдите на официальный сервер визуализации: **[https://www.plantuml.com/plantuml/uml/](https://www.plantuml.com/plantuml/uml/)**.
3. Вставьте скопированный код в левое текстовое поле. Диаграмма сформируется в правой части экрана. Для сохранения используйте ссылку «PNG» под областью вывода.

### Вариант 2: Визуализация в среде VS Code
1. Установите редактор Visual Studio Code и расширение **PlantUML** (автор: jebbs).
2. Откройте требуемый `.puml` файл и нажмите комбинацию клавиш `Alt+D` для вызова панели предпросмотра.
3. Через контекстное меню выберите пункт **Export Current Diagram** для сохранения файла в формате PNG/SVG.

## 8 Типичные неисправности и методы их устранения

### Ошибка: «A network-related or instance-specific error occurred while establishing a connection to SQL Server»
* **Причина**: Служба SQL Server не запущена, либо в файле `App.config` указано неверное имя хоста.
* **Устранение**: Откройте оснастку «Службы» (`services.msc`) и убедитесь, что служба **SQL Server (SQLEXPRESS)** находится в статусе «Выполняется». Проверите корректность строки подключения.

### Ошибка: «Cannot open database "uniqueness_db" requested by the login»
* **Причина**: Целевая база данных не создана на сервере.
* **Устранение**: Повторно выполните инструкции Раздела 2, запустив скрипт `create_database.sql` через SSMS.

### Ошибка компиляции: «The name 'BCrypt' does not exist in the current context»
* **Причина**: В проекте отсутствует библиотека хеширования паролей.
* **Устранение**: Выполните повторную установку пакета через Package Manager Console, как описано в Разделе 5.
